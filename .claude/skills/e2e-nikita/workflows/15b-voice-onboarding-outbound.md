## Phase G: Outbound Call Delivery (~5 scenarios)

Return to the Ch3 active test user for all Phase G scenarios.

### S-15.7.1: Create scheduled_events row with platform='voice' [NEW]
- Priority: P1
- Method: A (API)
- Steps:
  1. Call a task or internal endpoint that schedules a voice follow-up, OR SQL-insert directly:
     ```sql
     INSERT INTO scheduled_events (user_id, platform, event_type, content, scheduled_at, status)
     VALUES ('<USER_ID>', 'voice', 'call_reminder',
       '{"voice_prompt":"Hey, just thinking about you","agent_id":"default"}',

     ```
  2. Verify row exists
- Pass criteria: `SELECT COUNT(*) FROM scheduled_events WHERE user_id='<USER_ID>' AND platform='voice' AND status='pending';` — count >= 1

### S-15.7.2: Deliver cron processes voice event — initiates call [NEW]
- Priority: P1
- Method: A (API)
- Preconditions: S-15.7.1 completed, user has `phone` set
- Steps:
  1. Ensure `users.phone` is set: `UPDATE users SET phone='+41787950009' WHERE id='<USER_ID>';`
  2. `curl -X POST $API/tasks/deliver -H "Authorization: Bearer $TASK_AUTH_SECRET"`
- Pass criteria: HTTP 200; ElevenLabs dashboard shows new outbound call initiated (or error logged if no ElevenLabs creds in test env — acceptable with log evidence)

### S-15.7.3: job_executions row created for deliver task [NEW]
- Priority: P1
- Method: A (API)
- Steps: After S-15.7.2
- SQL verify: `SELECT status, executed_at FROM job_executions WHERE job_name='deliver' ORDER BY executed_at DESC LIMIT 1;`
- Pass criteria: Row exists with `status='success'` or `status='completed'`, `executed_at` within last 2 minutes

### S-15.7.4: Gap-based trigger — 24h text silence creates voice touchpoint [NEW]
- Priority: P2
- Method: S+A (SQL-Setup + API)
- Steps:
  1. `UPDATE users SET last_interaction_at = NOW() - INTERVAL '25 hours' WHERE id='<USER_ID>';`
  2. `curl -X POST $API/tasks/proactive-outreach -H "Authorization: Bearer $TASK_AUTH_SECRET"` (or equivalent gap-check task)
  3. Wait 5s
- SQL verify: `SELECT platform FROM scheduled_events WHERE user_id='<USER_ID>' ORDER BY created_at DESC LIMIT 1;`
- Pass criteria: New `scheduled_events` row with `platform='voice'` created

### S-15.7.5: Conflict suppression — no outbound during active conflict [NEW]
- Priority: P2

     ```sql

     WHERE id='<USER_ID>';
     ```
  2. Trigger outbound scheduling (same as S-15.7.4)
- Pass criteria: No new `scheduled_events` row with `platform='voice'` created during active conflict (temperature > 0.3 threshold)
