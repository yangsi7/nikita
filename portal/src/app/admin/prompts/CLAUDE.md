# portal/src/app/admin/prompts/ — Generated Prompt Inspector

## Purpose

Admin-only viewer for prompts generated by the 11-stage pipeline's `prompt_builder` stage (see `memory/architecture.md` §"11-Stage Async Pipeline"). Lets reviewers inspect the EXACT prompt that was sent to the LLM for a given conversation turn — useful for debugging persona drift, vice-injection issues, vulnerability gating, and chapter-context bleed.

## Key Files

- `page.tsx` — server-component shell wrapping the client component.
- `page-client.tsx`:
  - `useQuery(["admin", "prompts", page], () => adminApi.getPrompts(page))` paginated list (`:21-25`).
  - `useQuery(["admin", "prompt", selectedId], () => adminApi.getPrompt(selectedId!))` detail-on-select (`:27-31`).
  - Sheet/drawer pattern (`shadcn/ui Sheet`) for prompt-detail rendering with `ScrollArea` for long prompts.
  - `LoadingSkeleton` / `ErrorDisplay` / `EmptyState` shared components.
  - Stale time from `STALE_TIMES.admin` constant (`@/lib/constants`).

## Callers

- Admin users navigating from admin index.
- Linked from individual conversation viewers (e.g., `/admin/conversations/[id]`) to inspect the exact prompt for a flagged turn.

## Gotchas

- **Admin-only auth gate**: middleware should restrict `/admin/*` routes to users with `is_admin=true`. Verify in `portal/middleware.ts` (or `proxy.ts` if migrated). Do NOT rely on client-side checks.
- **Token-heavy responses**: full prompts can be 4-8K tokens; the `Sheet` component's `ScrollArea` is required to keep the viewport sane.
- **`adminApi.getPrompts`** lives in `portal/src/lib/api/admin.ts` and hits the FastAPI backend `/api/v1/admin/prompts` endpoint (verify endpoint path on master).
- **Pagination state in URL is OPTIONAL**: current implementation keeps `page` in component state (resets on remount). Consider promoting to `useSearchParams` for shareable links if needed.
- **No mutation paths here** — pure read-only viewer. Mutations live in `/admin/prompts-test/` or similar (verify; not in current scope).
- **Prompt redaction**: prompts may include user PII (names, locations) — admin is the only role gated to see it. Be careful copy/pasting prompt content into Slack/issues.
