## T0.2: Root cause

**File**: `nikita/agents/onboarding/tools/web_search.py:173-178`

```python
user_location = WebSearchUserLocation(
    city=city,
    country=country,
    region="",        # <-- BUG: empty string rejected by Anthropic API
    timezone="",      # <-- BUG: same
)
```

Comment at line 171-172 ("Anthropic accepts partial location") is incorrect.
"Partial" means OMIT optional fields, not pass empty strings. `WebSearchUserLocation`
is `TypedDict(total=False)` — all fields optional; empty-string values trip the
Anthropic API validator (`web_search_20250305` builtin tool).

Pydantic AI 1.71+ docs:
- `WebSearchUserLocation` is a TypedDict, total=False
- city + country are sufficient for localization
- region + timezone are optional and should be omitted when not known

## Hypothesis verification

The `tools[6]` index in the API error message corresponds to the WebSearchTool
appended via `agent.run(..., builtin_tools=[prepared_web_search(ctx)])`. After 5
firecrawl tools + 1 cohort tool, WebSearchTool is at index 6.

When city="zurich" / "berlin" / "nyc" and detect_country returns a known ISO code,
`user_location` is constructed with empty `region`/`timezone` — and this is exactly
what Walk B5 hit (user typed "Zurich" as city in step 9).

## Fix

Omit `region` + `timezone` from the dict entirely when not populated:

```python
user_location = WebSearchUserLocation(city=city, country=country)
```

(or build kwargs dict conditionally for forward-compat with future region/timezone
detection).

## Out-of-scope

- The 30s "timeout" wall-clock UX is a downstream symptom of the API rejection
  cascading through agent.run retries; once the API call succeeds, latency drops
  below the existing budget.
- Backstory generation pipeline itself (archetype picker → persona generator) is
  upstream-correct; only the WebSearchTool config feeding the agent is broken.
